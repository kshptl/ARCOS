# DEA Diversion Control — probe notes

**Probed on:** 2026-04-30
**Source index:** https://www.deadiversion.usdoj.gov/pubs/reports/index.html

## Probe outcome

`https://www.deadiversion.usdoj.gov/pubs/reports/index.html` returns
HTTP 404 (the site has apparently reorganized since the spec was
written). `https://www.deadiversion.usdoj.gov/pubs/` responds 200 but
the payload is identical byte-length to the 404 template — the site is
a Single-Page-App-ish catch-all HTML template. Determining the current
canonical URLs for annual statistical PDFs requires either (a)
browsing the site interactively or (b) searching DOJ FOIA archives,
both outside this session's scope.

Per the plan's spike-handling rules, we therefore:

1. Captured hand-authored **synthetic PDF fixtures** under
   `pipeline/tests/fixtures/dea/diversion_20{12,14}_sample.pdf` that are
   shape-compatible with what a real DEA annual report prose page looks
   like (regex-extractable counts + notable case titles).
2. Kept `notebooks/03-dea-probe.py` with placeholder `REPLACE_WITH_…`
   URLs. A maintainer with a browser MUST overwrite these with real
   URLs from https://www.deadiversion.usdoj.gov/ before the first
   production fetch, then re-run the probe and `test_clean_dea.py`.

The synthetic PDFs were generated by a minimal hand-rolled PDF writer
(see git history for `notes/dea.md`) so pdfplumber can extract text
from them. Content includes placeholder counts (`1,245` / `1,418`),
placeholder notable-case titles (e.g. `United States v. Tug Valley
Pharmacy`), and an `Operation …` phrase. They are **NOT real DEA
Diversion reports** and must not be published.

## Extraction strategy

- We extract **enforcement action counts** from the summary prose
  (typically early pages) via regex (e.g. `1,245 administrative actions`)
- We extract **notable case titles** (`United States v. …`, `Operation …`)
  via best-effort regex
- PDFs change format across years; the parser tolerates (a) table-based,
  (b) prose-only, (c) mixed layouts — only prose is exercised here

## Known quirks

- Some years include a separate "Cases of Interest" subsection — we extract from there if present
- Counts are sometimes "approximately 1,250" — we parse the integer out or fall back to 0 + log a warning
- Content hashes shift year-over-year even for "the same" report — we don't canary DEA, we just recompute
